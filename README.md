🚀 Mobility Cloud System: Telemetry Management GuideCQRS 패턴을 기반으로 설계된 차량 텔레메트리 실시간 관리 시스템입니다. 대량의 차량 데이터를 실시간으로 수신(Command)하고, 효율적인 조회(Query)를 위해 데이터 저장소와 처리 경로를 분리하여 고가용성을 확보했습니다.🏗️ System Architecture & Data Flow시스템은 책임 분리를 위해 **Command(쓰기)**와 Query(읽기) 영역으로 나뉘며, Kafka를 통해 비동기적으로 데이터를 동기화합니다.📥 [Command] 데이터 수신 및 저장Ingest API (FastAPI): 차량 시뮬레이터로부터 POST /api/query/telemetry 요청을 통해 상태 데이터를 수신합니다.Kafka Producer: 수신 데이터를 car.telemetry.events 토픽으로 즉시 발행합니다.MongoDB Consumer: 전체 이력(telemetry_history)을 영구 저장하여 데이터 영속성을 보장합니다.📤 [Query] 데이터 조회 및 캐싱Redis Consumer: Kafka 이벤트를 구독하여 차량별 최신 상태(vehicle:*:latest)를 Redis에 업데이트합니다 (TTL 60s).Query API (FastAPI): GET /api/vehicles 요청 시, DB가 아닌 Redis 캐시에서 데이터를 읽어 초고속 응답을 제공합니다.Frontend (React): Leaflet 지도를 활용하여 실시간 차량 위치를 시각화합니다.☸️ Kubernetes Infrastructure (Helm)k8s-manifests/ 디렉토리에는 서비스 성격에 따른 4개의 Helm 차트가 포함되어 있습니다.레이어컴포넌트상세 설명InfraKafka, MongoDB, RedisStatefulSet 기반 운영 및 데이터 보존 정책 적용CommandIngest API, ConsumerHPA(Horizontal Pod Autoscaler)를 통한 수평 확장 지원QueryQuery API, Consumer읽기 전용 서비스 배포FrontendReact, Nginx IngressVWorld Proxy 설정 및 경로 기반 라우팅Security Note: VWorld 지도 API 키는 Nginx Proxy를 통해 은닉 처리되어 클라이언트에 노출되지 않습니다.📊 Monitoring & OperationsPrometheus & Grafana: 시스템 메트릭 수집 및 대시보드 시각화.Alertmanager: 리소스 부하 및 파드 장애 시 Discord를 통해 실시간 알림 전송.Data Persistence: NFS Storage(10.0.2.100) 마운트를 통해 파드 재시작 시 데이터 보존.CronJob: MongoDB(15분 주기 이력 정리), Redis(10분 주기 고아 키 정리) 자동 관리.🛠️ CI/CD & DeploymentCI/CD: Jenkinsfile을 통한 빌드 → Harbor 이미지 푸시 → Helm 배포 자동화.Manual Deploy: docs/deploy.sh를 통해 로컬/스테이징 환경 수동 배포 지원.Secret Management: secret-values.yaml을 통해 보안 민감 정보를 분리 관리합니다.
